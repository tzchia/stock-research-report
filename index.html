01方法：事件 → 括號 → 勝率

延續原目的（找出值得跟的分點），但反轉順序：先標記大動作，再媒合買賣，最後評分點。 每一 (分點, 股票, 日) 取淨額 = 買 − 賣；同時滿足 |淨額| ≥ 1% × 當日量 且 |淨額| × 日VWAP ≥ NT$5M 即為一個事件——淨買是開括號 (，淨賣是閉括號 )。

事件流（同一分點×同一股票，時間序）： B₁ 300張 ─── B₂ 500張 ─── S₁ 800張 堆疊：push B₁ → push B₂ → S₁ 先吃頂端 B₂ 500 張（LIFO），再吃 B₁ 300 張 配對：(B₂ 500張 )S₁ 與 (B₁ 300張 )S₁ ── 量可部分拆分 空堆疊遇到 ) → 「無效閉括號」＝窗前庫存或閾值下累積，列為未媒合賣量 收盤仍留在堆疊的 ( → 未平倉，以窗末 VWAP 按市值計，另列不入勝率
  • 報酬：買日 VWAP → 賣日 VWAP，費稅後（手續費 0.1425% × 2 + 證交稅 0.3%）。 Fubon 無逐價明細 ⇒ 以 VWAP 計價，量測的是「選日能力」，日內執行優劣不可見。
  • 基準：每組配對持有 h 個交易日，就跟同一檔股票全期間所有 h 日視窗比—— 超額 = 配對報酬 − 視窗平均；隱含勝率 = 視窗為正的比例。 不做這步，2023–25 多頭會讓所有「先買後賣」看起來都會贏。
  • 顯著性：二項檢定會被重疊配對灌水（同一大買單分次出貨 = 偽複製）， 改用 circular-shift 置換檢定——整個分點的配對視窗共同隨機平移後重算， 保留內部重疊結構，讓虛無分布誠實定價。BH-FDR 控多重檢定。

06驗證

  • 演算法：媒合/基準/置換皆有手算 fixture 單元測試（tests/test_bracket.py，9 項全綠）。
  • 數字：獨立 agent 依文字規格從頭重寫程式對帳——全域件數（事件、配對、媒合率、分點數）完全相等、 抽查分點 14/14 條通過（最大相對誤差 0.06%）。個股帳本與分點層級共用同一批配對，僅分組鍵不同。
  • 紅隊抓到兩個真缺陷，已修： (1) 置換偏移原以大整數對各 (股,持有期) 群組分別取模——群組偏移實質獨立、破壞跨群組相關， 虛無下 α=5% 實測假陽性 24.5%；改為「交易日空間」共同偏移後回到校準帶。 (2) DuckDB 平行分組使迭代順序不穩、p 值不可重現；改為 per-unit RNG + 完整排序鍵，同 seed 逐位可重現。
  • 統計校準：合成虛無資料（200 家假單位 × 2023–25 真實 VWAP 序列）：獨立擺放虛無 α=5% 實測 4.0%（超額）/ 2.0%（勝率，保守側）、FDR≤10% 家數 0；極端叢集虛無 8.5% / 6.5%、FDR 家數 1.5% / 0%。機制層級結論適用於本 5 年窗。 單股帳本恰為校準實驗中最乾淨的情境（單一股票、單一時間軸——「B2 單群組」variant 實測 α=5% 為 4.5%，恰在名目值附近）。
